# Архитектура BitTorrent Client

## Обзор системы

BitTorrent Client состоит из нескольких слоёв, каждый из которых отвечает за определённый аспект функциональности.

## Слои архитектуры

### 1.层 User Interface (UIManager)
**Файлы:** UIManager.h/cpp

Отвечает за взаимодействие с пользователем:
- Создание и управление элементами управления (ListView, ProgressBar, Buttons)
- Обработка пользовательского ввода
- Обновление отображения информации о загрузках
- Форматирование и отображение данных (размеры, скорость, время)

**Ключевые методы:**
- `Initialize()` - инициализация UI
- `UpdateUI()` - обновление информации на экране
- `OnCommand()` - обработка команд пользователя
- `HandleFileOpen()` - выбор торрент-файла

### 2. Core Business Logic (BitTorrentClient)
**Файлы:** BitTorrentClient.h/cpp

Главный координатор приложения:
- Управление жизненным циклом приложения
- Координация между всеми менеджерами
- Управление активными загрузками
- Предоставление единого интерфейса для UI

**Ключевые методы:**
- `Initialize()` - инициализация клиента
- `StartDownload()` - начало загрузки торрента
- `PauseDownload()/ResumeDownload()` - управление загрузками
- `Shutdown()` - корректное завершение

### 3. Download Management (DownloadManager)
**Файлы:** DownloadManager.h/cpp

Управление процессом загрузки:
- Контроль скорости загрузки/выгрузки
- Отслеживание прогресса
- Расчет времени завершения
- Поддержка паузы/возобновления

**Состояния:**
- IDLE - неактивно
- DOWNLOADING - загружается
- PAUSED - приостановлено
- COMPLETED - завершено
- STATE_ERROR - ошибка

### 4. Torrent Parsing (TorrentCore)
**Файлы:** TorrentCore.h/cpp

Работа с .torrent файлами:
- Парсинг Bencode формата
- Извлечение информации о торренте
- Вычисление хешей информации
- Управление метаданными кусков

**Структуры данных:**
- `TorrentInfo` - информация о торренте
- `FileInfo` - информация о файлах
- `PieceInfo` - информация о кусках

### 5. Tracker Communication (TrackerManager)
**Файлы:** TrackerManager.h/cpp

Взаимодействие с трекерами:
- Запросы к трекерам
- Получение списка пиров
- Управление информацией о сидах/личах
- Расчет интервалов обновления

**Структуры:**
- `PeerInfo` - информация о пире
- `TrackerManager` - работа с трекерами
- `PeerManager` - управление пирами

### 6. Storage Management (StorageManager)
**Файлы:** StorageManager.h/cpp

Управление локальным хранилищем:
- Сохранение загруженных данных
- История загрузок
- Метаданные торрентов
- Структура директорий

**Основные методы:**
- `SaveDownloadHistory()` - сохранение истории
- `LoadTorrentMetadata()` - загрузка метаданных
- `GetTorrentDataPath()` - путь к данным торрента

### 7. Cache Management (CacheManager)
**Файлы:** CacheManager.h/cpp

Кеширование данных:
- Управление кусками данных в памяти
- Контроль размера кеша
- Верификация целостности кусков
- Управление блоками внутри кусков

**Классы:**
- `CacheManager` - управление кешем
- `BlockManager` - управление блоками

### 8. Settings Management (SettingsManager)
**Файлы:** SettingsManager.h/cpp

Управление конфигурацией:
- Загрузка/сохранение настроек
- Отслеживание статистики
- Управление параметрами приложения

**Компоненты:**
- `SettingsManager` - настройки приложения
- `StatisticsManager` - статистика использования

### 9. Logging (Logger)
**Файлы:** Logger.h/cpp

Система логирования:
- Логирование событий разных уровней
- Сохранение логов в файл
- Отладка и мониторинг

**Уровни логирования:**
- DEBUG - подробная информация для отладки
- INFO - информационные сообщения
- WARNING - предупреждения
- LOG_ERROR - ошибки
- CRITICAL - критические ошибки

## Паттерны проектирования

### 1. Singleton Pattern
- **Logger** - единственный экземпляр логгера в приложении

### 2. Manager Pattern
- **DownloadManager** - управляет процессом загрузки
- **TrackerManager** - управляет взаимодействием с трекерами
- **StorageManager** - управляет хранилищем
- **SettingsManager** - управляет настройками

### 3. Factory Pattern
- Создание объектов `TorrentInfo` через `TorrentParser`

### 4. Observer Pattern
- UIManager подписывается на обновления от DownloadManager

## Жизненный цикл приложения

```
1. Инициализация
   ├─ Создание BitTorrentClient
   ├─ Инициализация StorageManager
   ├─ Загрузка настроек
   ├─ Загрузка статистики
   └─ Инициализация UI

2. Основной цикл
   ├─ Таймер обновления UI (каждые 500мс)
   │  ├─ Обновление прогресса
   │  ├─ Расчет скорости
   │  └─ Отображение информации
   ├─ Обработка пользовательского ввода
   └─ Работа фоновых потоков

3. Завершение
   ├─ Остановка загрузок
   ├─ Сохранение настроек
   ├─ Сохранение статистики
   └─ Очистка ресурсов
```

## Многопоточность

- **Главный поток** - обработка UI и основная логика
- **Поток загрузки** - фоновая загрузка (в DownloadManager)

Потокобезопасность обеспечивается:
- Использованием `std::mutex` в DownloadManager
- Атомарными переменными для флагов
- Безопасным обменом данными между потоками

## Производительность

### Оптимизации:

1. **Кеширование** - часто используемые данные хранятся в памяти
2. **Ленивая загрузка** - данные загружаются по мере необходимости
3. **Пулинг потоков** - одного потока загрузки достаточно для большинства случаев
4. **Эффективное управление памятью** - автоматическая очистка старых данных

### Масштабируемость:

- Система поддерживает большое количество одновременных загрузок
- Гибкая система управления пирами
- Адаптивный контроль скорости

## Обработка ошибок

1. **Уровень парсинга** - проверка корректности .torrent файлов
2. **Уровень сетевой** - повторное подключение при разрыве соединения
3. **Уровень хранилища** - проверка целостности файлов
4. **Уровень UI** - информирование пользователя об ошибках

## Безопасность

- Валидация входных данных
- Проверка целостности через хеши
- Безопасное управление памятью
- Логирование подозрительной активности
